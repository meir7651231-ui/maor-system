# מאור החסד — מערכת ניהול

מערכת ניהול מלאה לעמותה: משפחות ובני משפחה, חוגים ושיבוצים, כרטיסיות ותשלומים,
נוכחות והיעדרויות, לוח שנה עברי-לועזי, יומני חדרים, תורמים ותרומות, דוחות והגדרות.

בנייה חדשה ונקייה (React 18 + TypeScript + Vite) של אב-הטיפוס `MaorSystemClean.html`.

## ארכיטקטורה

- **אתר סטטי לחלוטין** — אין שרת, אין חשבונות צד-שלישי, אין עלות חודשית.
  ההחלטה התקבלה אחרי השוואת שלוש חלופות (מקומי-בלבד / Firebase / שרת עצמאי)
  לפי הקריטריון: מסירה חד-פעמית ללקוח ואפס תחזוקה לאורך שנים.
- **הנתונים נשמרים אצל הלקוח** בשלוש שכבות:
  1. `localStorage` — שמירה אוטומטית חצי שנייה אחרי כל שינוי.
  2. `IndexedDB` — עותק שני + טבעת צילומים יומיים (30 ימים אחורה) לשחזור עצמי.
  3. **קובצי גיבוי JSON** — הורדה ידנית בלחיצה + גיבוי אוטומטי בסוף כל יום עבודה.
- שחזור מלא מכל אחת מהשכבות דרך מסך ההגדרות.


## פיתוח

```bash
cd maor
npm install
npm run dev        # שרת פיתוח
npm run typecheck  # בדיקת טיפוסים
npm test           # בדיקות יחידה
npm run build      # בנייה ל-dist/
```

## פריסה (Deploy)

הפלט הוא תיקיית `dist/` סטטית — אפשר להעלות לכל אחסון סטטי:

- **GitHub Pages** (מומלץ — חינם, ללא כרטיס אשראי): להעלות את `dist/` לענף
  `gh-pages` או להגדיר Actions. הנתיבים יחסיים (`base: './'`) כך שהאתר עובד
  מכל תת-נתיב.
- Cloudflare Pages / Netlify — גרירת `dist/` לממשק.

## מה חשוב למסור ללקוח (בכתב)

1. הנתונים נשמרים **בדפדפן של המחשב** — לעבוד תמיד מאותו מחשב ואותו דפדפן.
2. המערכת מורידה **קובץ גיבוי אוטומטי בסוף כל יום** — לשמור את הקבצים בתיקייה
   מסודרת (רצוי מסונכרנת לענן, למשל Google Drive).
3. מעבר מחשב = ייצוא גיבוי במחשב הישן ← ייבוא במסך ההגדרות במחשב החדש.
4. אין להשתמש במצב גלישה פרטית ואין לנקות נתוני אתרים לדומיין של המערכת.
